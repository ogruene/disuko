# Benutzer-Verwaltung

## Benutzerrollen:

- Der **Eigentümer** erstellt und verwaltet das Projekt, seine Metadaten und seine Projektversionen, kann anderen
  Benutzern Zugriff auf das Projekt gewähren und ist auch in der Lage, API Token zu erstellen, zu aktualisieren oder zu
  widerrufen; Der Eigentümer hat auch das Recht, Daten aus dem Projekt anzuzeigen, hoch- und herunterzuladen. Darüber
  hinaus kann der Eigentümer Überprüfungen und Genehmigungen für das Projekt anfordern. Aufgrund ihrer Berechtigungen können nur Benutzer mit internem Zugriffsbereich als Eigentümer ausgewählt werden.
- Der **Lieferant** erstellt und verwaltet Projektversionen und ist auch in der Lage, API Token zu erstellen, zu
  aktualisieren oder zu widerrufen; Der Lieferant hat auch das Recht, Daten aus dem Projekt einzusehen, hoch- und
  herunterzuladen.
- Ein **Betrachter** hat das Recht, Daten aus dem Projekt anzuzeigen und herunterzuladen.

## Anmerkungen:

- Nur interne Benutzer können Projekte erstellen
- Nur interne Benutzer können zu Eigentümern gemacht werden
- Externe Benutzer können nur auf Projekte zugreifen, für die ihnen eine Rolle zugewiesen wurde
