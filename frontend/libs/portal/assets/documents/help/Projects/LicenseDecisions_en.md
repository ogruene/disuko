# License Decisions

Some FOSS components are offered under different licensing options. This means that the **project responsible** gets to make a choice which license is best. It is possible to document this choice in FOSS Disclosure Portal to evaluate future SBOM deliveries based on this decision.

## What do I have to do?

- Put simply, the decision which option will be used needs to be made within the project and then properly documented within the FOSS Disclosure Portal.
- Before a license choice is made, the component overview always shows the worst-case status. In the „Decision“ column you can see a blue “i” icon if it is possible to decide between licenses.
- If you are presented with a choice of licenses, use the most permissive and known license. If you really must use a copyleft license to resolve a license compatibility issue, please be aware of the consequences and take appropriate action to mitigate these.
- The “project responsible” can then document which license has been selected within the Disclosure Portal directly so that the displayed policy status reflects this.
- Info: You can identify the “Project Responsible” by looking in the user management tab of the project. This is the only person on the project who is offered this functionality.

## Where can I see if a decision has been made, or change a decision?

- In the project overview, you can see any license decisions that have been made for this project in the “License Decisions” tab.
- To revise a decision, the project responsible can use the cancel function shown as an action. The decision can still be seen in this overview but it now has a yellow cancelled status.
- Then you can make a new decision on the component overview.
