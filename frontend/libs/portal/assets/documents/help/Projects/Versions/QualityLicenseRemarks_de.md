# <i class="v-icon notranslate material-icons mdi mdi-gavel primary--text"></i> Lizenzbemerkungen

- **Geltungsbereich:** Lizenzbemerkungen enthalten eine Zusammenfassung von Verpflichtungen, die sich aus den Lizenzen der Komponenten ergeben, auf die in der aktuellsten SBOM-Lieferung verwiesen wird.
- **Warnstufen:**
  - <i class="v-icon notranslate v-icon--dense material-icons mdi-bell mdi v-theme--dark v-icon--size-default text-red"></i> **Alarm:** mindestens eine Lizenz innerhalb der SBOM enthält eine Klassifizierung, die das Potenzial hat, ein besonders hohes Risiko darzustellen oder anderweitig besonders problematisch ist. Es ist wichtig, die Klassifizierungen, die den Alarm ausgelöst haben, im Zusammenhang mit dem vorliegenden Anwendungsfall zu überprüfen. Alarme sollten als Blocker behandelt werden, wenn keine mildernden Umstände vorliegen.
  - <i class="v-icon notranslate material-icons mdi-alert mdi v-theme--dark v-icon--size-default text-warning" style=""></i> **Warnung:** Mindestens eine Lizenz innerhalb der SBOM enthält eine Klassifizierung, die zu einer Verpflichtung führen kann, die schwer zu erfüllen oder unerwartet sein könnte. Es lohnt sich, diesen Warnungen besondere Aufmerksamkeit zu schenken und sie im Zusammenhang mit der geplanten Softwareauslieferung zu betrachten. Wenden Sie sich im Zweifelsfall an die Rechtsabteilung.
  - <i class="v-icon notranslate v-icon--dense material-icons mdi-information-outline mdi v-theme--dark v-icon--size-default text-textColor"></i> **Information:** Mindestens eine Lizenz innerhalb der SBOM enthält eine Klassifizierung, die als typisch für Open Source Lizenzen eingestuft wurde und erfüllt werden sollte. Der Informationsstand ist ein nützlicher Anhaltspunkt, um schnell zu erkennen, welche Verpflichtungen für die SBOM als Ganzes gelten.
