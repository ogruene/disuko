# User Management

## User Roles:

- The **Owner** creates and manages the project, its metadata and its project versions, can grant other users access to
  the project, and is also able to create, refresh or revoke API tokens; the owner also has priviledge to view, upload
  and download data from the project. Furthermore, the owner is able to request checks and approvals on the project. Due to their priviledges, only users with internal access scope can be selected as owner.
- The **Supplier** creates and manages project versions and is also able to create, refresh or revoke API tokens; the
  supplier also has priviledge to view, upload and download data from the project.
- A **Viewer** has priviledge to view and download data from the project.

## Remarks:

- Only internal users are able to create projects
- Only internal users can be made project owners
- External users are only able to access projects, on which they have been assigned a role
